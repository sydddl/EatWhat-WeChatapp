# 功能说明

## 产品定位

EatWhat 是给固定饭搭子使用的微信群“吃什么抽签”工具。推荐用法是一个固定小组长期维护同一个学校周边餐厅池，大家通过同一个 `groupId` 共享数据。

## 当前功能

### 固定小组

- 首页没有 `groupId` 时，不再自动创建随机小组。
- 用户可以创建一个固定小组，或输入朋友提供的 `groupId` 加入小组。
- 创建或加入后，`groupId` 会保存在本机缓存里。
- 首页可复制 `groupId`，也可以切换小组。
- 分享路径会携带 `groupId`，群成员从分享卡片打开后进入同一个小组。

### 餐厅管理

- 查看当前小组的餐厅列表。
- 添加餐厅。
- 编辑餐厅。
- 禁用/启用餐厅。
- 禁用餐厅不会参与抽签，但仍保留在列表里。

餐厅字段：`name`、`address`、`priceRange`、`tags`、`baseWeight`、`sourceUrl`、`note`。

### 大众点评文本解析

添加餐厅页可以粘贴大众点评分享文本，本地解析店名、地址和 `sourceUrl`。当前是启发式解析，不接入大众点评 API。

### 加权抽签算法

先用筛选条件得到候选集合：

```latex
\[
C = \{ r_i \mid r_i \in R,\; r_i.disabled = false,\; r_i \text{ matches filters} \}
\]
```

每个候选餐厅使用非负权重：

```latex
\[
w_i = \max(0,\; r_i.baseWeight)
\]
```

餐厅 `r_i` 被抽中的概率是：

```latex
\[
P(r_i) = \frac{w_i}{\sum_{r_j \in C} w_j}
\]
```

实现时生成区间随机数：

```latex
\[
u \sim U(0,\; \sum_{r_j \in C} w_j)
\]
```

然后按候选餐厅顺序累减权重，第一次使剩余值小于等于 0 的餐厅就是结果。权重越高，被抽中的概率越高；候选为空时不会写入历史记录。

### 历史

每次抽签会写入 `draws` 集合，历史页展示最近 20 条。历史记录保存餐厅快照，后续编辑餐厅不会改掉旧抽签结果。

## 云端数据

云开发数据库集合：

- `groups`：固定饭搭子小组。
- `members`：加入小组的微信用户 openid。
- `restaurants`：小组共享餐厅池。
- `draws`：抽签历史记录。

云函数：`createGroup`、`joinGroup`、`addRestaurant`、`listRestaurants`、`drawRestaurant`。
### 喜好代币

- 每个用户在一个小组内有 2 个加值代币和 2 个减值代币。
- 一个用户在同一家餐厅只能放一个代币；再次点击同类代币会取消，点击另一类会切换。
- 代币全组可见，餐厅卡片以徽章形式展示正负计数。
- 当前版本记录代币并写入抽签概率解释，但具体变动概率数值尚未校准，暂不改变最终权重。
- 需要新增云数据库集合：preferenceTokens。
